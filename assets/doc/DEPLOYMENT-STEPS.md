# Deployment Steps

## 1) Prerequisites

- Kubernetes cluster reachable (`kubectl cluster-info`)
- Helm installed (`helm version`)
- `assessment-platform` repo checked out

## 2) Prepare namespaces

```powershell
kubectl create namespace assessment-platform --dry-run=client -o yaml | kubectl apply -f -
kubectl create namespace assessment-jobs --dry-run=client -o yaml | kubectl apply -f -
```

## 3) Deploy chart

```powershell
helm dependency update .\helm

helm upgrade --install assessment .\helm `
  --namespace assessment-platform `
  --create-namespace `
  --set gitlab.enabled=true `
  --set gitlab.rootPassword="Admin1234!" `
  --set gitlabRunner.enabled=true `
  --set gitlabRunner.replicaCount=3 `
  --set backend.sso.enabled=false `
  --wait `
  --timeout 30m
```

Notes:
- GitLab image is hardcoded to `gitlab/gitlab-ce:16.7.0-ce.0`.
- Runner image is hardcoded to `gitlab/gitlab-runner:v16.7.0`.
- Runner registration token is auto-generated if not provided.
- Backend GitLab admin token is auto-generated by the Helm post-install/post-upgrade bootstrap job.

## 4) Verify platform health

```powershell
kubectl get pods -n assessment-platform
kubectl get deploy -n assessment-platform
kubectl get jobs -n assessment-platform
kubectl logs -n assessment-platform job/assessment-assessment-platform-gitlab-bootstrap
```

## 5) Verify runner auto-registration

```powershell
kubectl get pods -n assessment-platform -l app.kubernetes.io/component=gitlab-runner
kubectl logs -n assessment-platform deploy/assessment-assessment-platform-runner --tail=200
```

The logs should include `Runner registered successfully!`.
